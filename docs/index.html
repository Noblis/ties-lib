TIES Components

JSON Schema

TIES specifies a data interchange format for bidirectional sharing of data. The schema captures object items, relationships, and assertion information via a generic and flexible approach.

At its highest level, it consists of:

  • An array of exported multimedia object items (videos, images, documents) and associated meta-data, not including the raw multi-media data
  • An array of groups of related object items
  • An array of analyst annotations relating to object items included in the export
  • An array of supplemental descriptions relating to object items included in the export
  • Relationships between object items, annotations, and supplemental descriptions
  • An array of key/value pairs containing user-defined metadata
validator, converter, and data binding functionality
Validator

The TIES Schema Validator allows users to verify that TIES JSON documents are compliant with the current version of the TIES Schema. The validator provides error messages for any fields that do not conform to the schema. The validator can be used as a command-line tool, or invoked programmatically as part of the TIES Java and Python libraries.

Converter

The TIES Schema Converter allows users to convert TIES JSON documents that conform to older versions of the schema into documents that conform to the current version of the schema. The TIES Schema converter can be used as a command-line tool, or invoked programmatically as part of the TIES Java and Python libraries.

JSON Schema Data Binding

The Java and Python TIES Schema libraries provide a JSON Schema data binding implementation for the TIES Schema. The data binding tools allow users of the TIES libraries to deserialize TIES JSON documents into Java and Python data binding classes, modify the data in the data binding classes, and serialize the data binding classes as TIES JSON documents.
